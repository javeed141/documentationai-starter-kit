## Recent Releases

<Update label="2024-10-15" description="v2.1.0" tags={["feature", "bugfix", "improvement"]}>

## New Features

- Added support for reactive streams with `Flow.Publisher` integration
- Introduced configuration builder pattern for easier setup: `JavaConfig.builder().apiKey("YOUR_API_KEY").build()`

## Improvements

- Enhanced performance of async operations by `>30%` in high-throughput scenarios
- Better error messages with structured logging using `SLF4J` markers

## Bug Fixes

- Fixed memory leak in long-running HTTP clients
- Resolved race condition in multi-threaded event processing

</Update>

<Update label="2024-09-10" description="v2.0.0" tags={["feature", "breaking"]}>

## New Features

- Full GraphQL API support with schema introspection
- Native ARM64 compilation for Apple Silicon

## Breaking Changes

- Removed legacy `JavaClient` in favor of `JavaSDK` – update your imports to `com.java.sdk.JavaSDK`
- Default timeout increased from `30s` to `60s`; override explicitly if needed

## Bug Fixes

- Corrected JSON serialization for nested `Optional` fields
- Patched security vulnerability in JWT validation (CVE-2024-5678)

</Update>

## Upgrade Guide

Follow these steps to upgrade your Java integration to the latest version.

<Steps>
  <Step title="Update Dependencies" icon="download">

Use your build tool to update to the latest version.

<Tabs>
  <Tab title="Maven" icon="package">

````xml
<dependency>
    <groupId>com.java</groupId>
    <artifactId>java-sdk</artifactId>
    <version>2.1.0</version>
</dependency>
````

  </Tab>
  <Tab title="Gradle" icon="package">

````groovy
implementation 'com.java:java-sdk:2.1.0'
````

  </Tab>
</Tabs>

  </Step>
  <Step title="Review Breaking Changes" icon="alert-triangle">
    Check the release notes above for any required code migrations, such as updating deprecated methods.
  </Step>
  <Step title="Test Your Integration" icon="play-circle">
    Run your test suite and validate against `https://api.example.com` endpoints.
  </Step>
</Steps>

<Callout kind="alert">
Before upgrading in production, test in staging. Backup your configurations and monitor for `>24 hours` post-deployment.
</Callout>
